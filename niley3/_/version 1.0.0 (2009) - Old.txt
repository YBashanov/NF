Краткий обзор версий разработок
Номенклатура. 
A.BCD
A.B – версия
CD – доработка версии

Версия 1.0 
- сортировки страниц нет
- страницы в html, все в одной куче. Только html
- файла css нет. Стили – на страницах
- файлов javascript нет. Скрипты – на страницах

Версия 1.1
- сортировки страниц нет
- все страницы переведены в формат php. Чистого html нет!
- появился отдельный файл css
- появился отдельный файл js

Версия 1.2
- появилась сортировка страниц по папкам. Стили и js-скрипты в разных папках
- вынос функций в отдельные модули
- вынос кнопок и картинок в папку “@”
- использование чистого js для смены изображений

Версия 1.3
- правильное обращение к фоновому изображению при написании стиля объекта
- фоновое изображение – в папку со стилями
- использование чистого js в комплексе с библиотекой jquery. 

Версия 1.4
- разделение дизайна и информативной части. Папки pages и text
- динамическое создание таблиц по содержимому текстового файла
- генератор случайных чисел (случайных сообщений) на php
- использование прозрачности изображений
- использование капчи 

Версия 1.5
- использование технологии AJAX для отправки электронной почты
- использование технологии AJAX для электронных онлайн-часов
- сохранение директории cgi-bin. Были проблемы с удалением данной директории
- использование замедленного перехода с одной страницы на другую

Версия 1.6
- AJAX для вывода измененных изображений и сообщений (о неотправке почты, например). Изображения использовались в «золотых» часах
- файлы отправки электронной почты в отдельном файле
- использование jquery для создания анимации – появление или скрытие объектов, движение слайдов, вывод предупреждающих окон-alert, слайдшоу

Версия 1.7
- использование простых таблиц MySQL. 
- использование серверных переменных для создания статистики
- использование сеансовых переменных для обеспечения регистрации статистики
- использование файла __autoload для подгрузки одних и тех же скриптов и стилей для каждой страницы

Версия 1.8 (18.08.09)
- появился отдельный файл-функция натяжки дизайна (php + html) для автоматического генерирования страниц
- в каждой папке появился файл index.php, запрещающий пользователю свободно просматривать содержимое (путь) сайта
- использование корневого index.php для пересылки в папку pages к остальным страницам. Ссылка на главную из этой папки
- папки не вложены одна в другую, а находятся в корне сайта (за исключением вложенных в @), что облегчает перенос разработок на другой сайт
- не имеет смысла писать общий файл js, незачем подгружать его к каждой странице. Лучше – по частям.
