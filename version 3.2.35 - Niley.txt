-  версия Andrew 2.2 Koral (2013-01-21)
-------------------------------------------------------------------------------------------------------
Все преобразования касаются папок
- @ - графические и другие медиафайлы (для закачивания, скачивания и хранения)
- @/uploads - закачивание на сервер (пользовательские)
- @/uploads/video - то, что закачали пользователи (для просмотра)
- @/video - видео сервера (администратора, компании), то, что пользователи не закачивали, но это в доступе для просмотра (или хранения без просмотра)
Какую-то папку из директории @ можно закрыть с сервера файлом .htaccess, но это уже - закрытый доступ (платный) для скачивания
(не реализовано)
- ajax - для обработки асинхронного доступа к серверу
- backup - только для локальной машины 
- data - вся информация (текстовый контент для страниц без верстки)
-- /variables - переменные, доступные в любом месте страницы (global['var'], либо $var)
-- /variables_array - массивы переменных, доступные в любом месте страницы (global['arrays'], либо $arrays)
- error - все html-файлы для корректной обработки ошибок и выдачи информации пользователю
- js - классы javascript-файлов (в корне - пользовательские функции)
- js/classes - написанные мной готовые универсальные библиотеки javascript
- libraries - написанные мной готовые универсальные библиотеки php
- libraries/библиотека/index.php - путь до любой библиотеки (стандартный)
- modules - дополнительные модули, расширяющие функционал приложений
-- /config - настройки подключения к шаблонам, общие настройки, недоступные пользователю (в переменной global)
-- /global - определение глобальной переменной, подключение config
-- /includes - подключение модулей (для работы на каждой странице) и библиотек php-функций
-- /includesToAjax - аналогичное подключение модулей, но применительно для асинхронной обработки запросов
-- /includes_css - подключение css к странице
-- /includes_jsBottom - подключение javascript в конец шаблона страницы
-- /includes_jsTop - подключение javascript в начало шаблона страницы
-- /redirect_www - организация редиректа на www
-- /server.svg.js - включение настроек сервера (имени домена) для svg
-- /server_js - включение настроек сервера (имени домена) для javascript, а также gen20
-- /server_php - включение настроек сервера (имени домена) для php
- modules/authorization - определяет авторизацию пользователя
- modules/captcha - реализация капчи (неразрывно с modules/fonts)
- modules/content - функции работы с базами, для внедрения контента в страницу
- modules/db - база данных
-- /All_connection - полное подключение страницы к базе
-- /config - настройки подключения к базе
-- /Database - обычное подключение к базе (mysql)
-- /Database_i - подключение, использующее синтакс mysqli
-- /Error - обработчик пользовательских ошибок и исключений
-- /NewTables - список таблиц с расшифровками
- modules/flash - работа с flash
- modules/functions - функции, которые по некоторым причинам пока не вошли в мои библиотеки классов функций
- modules/session - логический обработчик посещений страницы
- modules/svg - работа с svg
- pages - файлы, содержащие коренные страницы, на которых динамически строятся остальные страницы
-- /_page - одинаковые включения для каждой страницы за исключением отображения (отображение может задаваться разными шаблонами)
-- /_pageAdmin - аналогичные включения для панели администратора (включений, как правило, меньше, чем в _pages)
- style - стили, не зависящие от шаблона отображения
- templates - шаблоны отображения
- templates/default - шаблон по умолчанию
-- /v_index - файл шаблона страницы index (в версии Cvett присутствует один файл шаблона для всех страниц - template_index)
функция, вызываемая в файле - template
- templates/default/html - все html-представления. Папка includes - для встраивания шаблонов из ajax
В корне - отображения для отдельных частей страницы
- templates/default/style - стили, применяемые внутри шаблона отображения

Корневые файлы для сервера
- .htaccess
- dumper (только для локальной машины)
- index
- startdate - 2013-01-21.txt - тут ведется учет перехода с версии на версию и особенностей разработки проекта
- version 2.2 - Koral.txt - текущая версия движка с описанием
